# Verbetervoorstel / Verbeterplan

## Doel

Leg concrete verbeteringen vast op basis van testresultaten en feedback.

## Structuur

-   Probleem: korte omschrijving
-   Impact: wie/waarom belangrijk
-   Oplossing: concreet voorstel
-   Prioriteit: hoog/middel/laag
-   Actie-eigenaar: wie voert uit

## Voorbeeld entry

-   Probleem: API accepteert te lange `player_name` waarden
-   Impact: UI breekt bij weergave
-   Oplossing: voeg validation-rule `max:50` en update UI
-   Prioriteit: middel
-   Eigenaar: developer X
